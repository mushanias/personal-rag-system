基于 FastAPI 与 Qdrant 的个人知识库 RAG 系统 

这是一个基于 Python 开发的个人知识库智能问答系统。项目采用 RAG 检索增强生成架构，对接 Moonshot API 实现智能交互，核心侧重于本地双通道检索优化、高性能内存管理以及接口级的数据安全隔离。
我的thinking文件中有着三个月编程的新手真实构建项目过程中的完整思考记录。包括为什么这样设计、遇到了什么问题、当时是怎么判断的。以及changelog上面的一些优化还有debug的思路希望对同水平的人有参考价值

=====================核心技术特性===================

1 三分类管理与安全隔离

知识体系划分为学习、生活、思考方法三大类别。
结合 FastAPI 路由与 Qdrant Payload Filter 联动，在数据库检索侧实现接口级的分类隔离与私密内容鉴权保护。

2 双通道混合检索 (Hybrid Search)

稠密向量：本地运行 sentence-transformers，采用 BAAI/bge-small-zh-v1.5 模型生成语义向量。
稀疏向量：使用 FastEmbed 生成 Qdrant/bm25 词频向量。
检索机制：双通道并行检索并持久化存储于 Qdrant，兼顾深度语义理解与关键词精准匹配。

3 服务性能优化

引入 FastAPI lifespan 机制，在服务启动时将 Embedding 模型与 Sparse 模型一次性加载至内存并常驻，消除请求触发时的模型重复加载开销。

4 双层防幻觉机制

检索层：设置严格的相似度分数阈值，自动截断低质量召回结果，防止无关文本干扰大模型。
生成层：编写强约束的 System Prompt，限制大模型仅能根据召回的上下文内容进行作答，并流式返回原文溯源供人工核查。

======================技术栈================================
核心框架：Python, FastAPI
向量数据库：Qdrant
文本表征：sentence-transformers, FastEmbed
大语言模型：Moonshot AI (Kimi) API
工程管理：Docker, Git

=========================项目运行=====================

1 环境配置
   创建环境配置文件，填入您的 MOONSHOT_API_KEY 以及 Qdrant 数据库的连接地址与端口。

2 数据灌入
   将知识库文档按分类放入指定目录，运行数据清洗与灌入脚本。系统会自动完成文本切块、生成双通道向量并写入 Qdrant。
3 服务启动
   项目支持 Docker 容器化部署。启动 FastAPI 服务后，即可通过不同的路由接口进行分类隔离的知识库问答。服务内置完整的异常处理与日志记录。
